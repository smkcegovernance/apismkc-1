## ?? Key Components

### 1. Service Account Credentials
**Location**: `GoogleDrive\service-account-credentials.json`

**Structure**:
```json
{
  "type": "service_account",
  "project_id": "your-project-id",
  "private_key_id": "...",
  "private_key": "-----BEGIN PRIVATE KEY-----\n...",
  "client_email": "service-account@project.iam.gserviceaccount.com",
  "client_id": "...",
  "auth_uri": "https://accounts.google.com/o/oauth2/auth",
  "token_uri": "https://oauth2.googleapis.com/token"
}
```

**Key Field**: `client_email` - Use this to share folders

### 2. Shared Folder Configuration
**Location**: `Web.config`

```xml
<add key="GoogleDrive_SharedFolderId" value="FOLDER_ID_FROM_URL" />
```

**How to Get Folder ID**:
1. Open folder in Google Drive
2. URL: `https://drive.google.com/drive/folders/1a2b3c4d5e6f7g8h9i0j`
3. Copy: `1a2b3c4d5e6f7g8h9i0j`

## ?? Why Shared Folder?

### The Problem
```
Service Account ? Upload to own drive ? ? ERROR!
                                         "No storage quota"
```

### The Solution
```
Service Account ? Upload to shared folder ? ? SUCCESS!
                  (in user's drive)           Uses user's quota
```

### Benefits
1. ? **No user interaction** - Fully automated
2. ? **No browser required** - Works on servers
3. ? **Centralized storage** - All files in one place
4. ? **Easy management** - Visible in Google Drive
5. ? **Simple backup** - Standard Google Drive backup
6. ? **Access control** - Manage via Google Drive sharing

## ?? Security Considerations

### ? Good Practices
- ? Use dedicated Google account for API storage
- ? Grant only **Editor** role (not Owner)
- ? Keep credentials JSON secure (don't commit to git)
- ? Rotate service account keys periodically
- ? Monitor folder access in Google Drive activity log

### ? Avoid
- ? Sharing personal Google Drive with service account
- ? Granting **Owner** role to service account
- ? Storing credentials in code or config files in git
- ? Using same account for multiple environments (dev/prod)
